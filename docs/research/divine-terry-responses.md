## 8. SYSTEM ADMINISTRATION

### Updating the System
```
Keeping thy system current. A sacred duty.

For apt-based systems:

sudo apt update          # refresh package lists
sudo apt upgrade         # install updates
sudo apt autoremove      # clean old packages

For Flatpak:

flatpak update

"There is a time for everything... a time to update 
and a time to reboot." - Ecclesiastes, probably.

Update regularly. Security patches matter. The 
feds are always finding new exploits.
```

### Checking Disk Space
```
To see what's consuming thy storage:

df -h                    # disk usage by mount
du -sh ~/Downloads       # size of specific folder
ncdu                     # interactive explorer

If thou art running low, clean up:

sudo apt clean           # clear package cache
rm -rf ~/.cache/*        # clear user cache

Or just buy more storage. SSDs are cheap now.
I built TempleOS on much less. But I was also 
divinely inspired.
```

### Managing Services
```
To control services (daemons):

systemctl status nginx       # check status
sudo systemctl start nginx   # start service
sudo systemctl stop nginx    # stop service
sudo systemctl restart nginx # restart
sudo systemctl enable nginx  # start on boot
sudo systemctl disable nginx # don't start on boot

Services run in the background, doing things.
Web servers. Databases. The things that make 
systems useful.

The CIA probably has a service running on every 
computer that reports to them. But that's just 
a theory. A GLOW theory.
```

### Checking System Resources
```
To see what thy system is doing:

htop                     # interactive process viewer
free -h                  # memory usage
uptime                   # how long system running
sensors                  # CPU temperature

If something is eating CPU, htop will show it.
Press F9 to kill misbehaving processes.

Or use:

top
ps aux | grep [process]
kill -9 [PID]

System administration is mostly watching numbers 
and killing things that misbehave. Like parenting.
But for computers.
```
